# 게임 기획서

## 1. 개요

- 프로젝트명: 몬스터 헌트 MVP
- 장르: 3D 액션 어드벤처
- 플랫폼: PC
- 엔진: Unity
- 목표 플레이 시간: 10분 내외의 반복 플레이
- 핵심 목표: 4개 난이도 구역을 탐험하며 몬스터를 사냥하고, 골드와 경험치를 획득해 10레벨까지 성장한다.

## 2. 콘셉트

플레이어는 하나의 사냥 필드 안에서 쉬움, 중간, 어려움, 보스 구역을 순서대로 공략한다. 기본 전투는 별도 무기 공격보다 이동 충돌, 박치기, 점프 밟기를 중심으로 구성한다. 조작은 단순하지만 몬스터의 위치, 체력, 접촉 방향, 점프 타이밍에 따라 사냥 효율이 달라지는 구조를 목표로 한다.

## 3. 핵심 재미

- 단순한 조작으로 바로 이해되는 몬스터 사냥
- 구역별 난이도 상승에 따른 성장 체감
- 몬스터 처치 보상으로 골드와 경험치 획득
- 레벨 1부터 10까지 짧은 성장 루프 제공
- CSV 데이터 수정만으로 몬스터 스탯과 보상 조정 가능

## 4. MVP 범위

MVP 포함 기능:

- 플레이어 이동, 점프, 박치기, 점프 밟기
- 몬스터 체력, 피격, 사망 처리
- 골드와 경험치 보상
- 레벨 1~10 성장
- 4개 구역 구성: 쉬움, 중간, 어려움, 보스
- 몬스터 CSV 데이터 로드
- 기본 HUD와 결과 화면

MVP 제외 기능:

- 장비 시스템
- 스킬 트리
- 인벤토리
- 퀘스트
- 복잡한 몬스터 AI
- 멀티플레이

## 5. 플레이 흐름

1. 타이틀 화면에서 게임을 시작한다.
2. 플레이어가 사냥 필드에 진입한다.
3. 쉬움 구역 몬스터를 사냥해 기본 조작과 보상을 익힌다.
4. 중간, 어려움 구역에서 레벨과 골드를 올린다.
5. 보스 구역에서 보스 몬스터를 처치한다.
6. 결과 화면에서 최종 레벨, 골드, 처치 수, 클리어 여부를 표시한다.

## 6. 구역 구성

| 구역 | 역할 | 권장 레벨 | 몬스터 방향 |
| --- | --- | ---: | --- |
| 쉬움 | 튜토리얼 및 초반 성장 | 1~3 | 느리고 체력이 낮은 기본 몬스터 |
| 중간 | 기본 전투 반복 | 3~6 | 이동 속도와 체력이 증가한 몬스터 |
| 어려움 | 위험도 상승 | 6~9 | 공격력과 추격 성능이 높은 몬스터 |
| 보스 | MVP 최종 목표 | 9~10 | 높은 체력과 강한 접촉 피해를 가진 보스 |

각 구역은 물리적으로 분리된 영역으로 구성한다. 초기 MVP에서는 구역 이동 제한을 강하게 두지 않고, 표지판이나 색상으로 난이도를 명확히 안내한다.

## 7. 조작

| 입력 | 기능 |
| --- | --- |
| WASD 또는 방향키 | 이동 |
| Space | 점프 |
| Left Shift 또는 마우스 좌클릭 | 박치기 |
| Esc | 일시정지 |

카메라는 3인칭 추적 카메라를 기본으로 한다. 마우스 회전은 MVP 이후 확장 항목으로 둔다.

## 8. 플레이어 시스템

### 이동

- 플레이어는 지면 위에서 상하좌우로 이동한다.
- 이동 중 몬스터와 충돌하면 접촉 방향과 상태에 따라 피해를 주거나 받는다.
- 이동 속도는 전 구역에서 동일하게 유지한다.

### 점프

- 지면에 닿아 있을 때만 점프할 수 있다.
- 점프 후 몬스터 머리 또는 상단 판정에 닿으면 점프 밟기로 처리한다.
- 점프 밟기 성공 시 플레이어는 짧게 튀어 오른다.

### 박치기

- 박치기는 짧은 시간 전방으로 돌진하는 공격이다.
- 박치기 중 몬스터 전방 또는 측면에 충돌하면 몬스터에게 피해를 준다.
- 박치기에는 쿨다운을 둔다.
- 보스에게도 피해를 줄 수 있지만 일반 몬스터보다 효율이 낮게 조정한다.

### 피해

- 플레이어가 공격 상태가 아닐 때 몬스터와 접촉하면 피해를 받는다.
- 피해를 받으면 짧은 무적 시간이 적용된다.
- 체력이 0이 되면 게임 오버 처리한다.

## 9. 전투 판정

| 상황 | 결과 |
| --- | --- |
| 플레이어가 이동 중 몬스터와 정면 충돌 | 플레이어 피해 |
| 박치기 상태로 몬스터와 충돌 | 몬스터 피해 |
| 점프 하강 중 몬스터 상단과 충돌 | 몬스터 피해 및 플레이어 반동 점프 |
| 무적 시간 중 몬스터와 충돌 | 추가 피해 없음 |
| 몬스터 체력이 0 이하 | 몬스터 처치 및 보상 지급 |

박치기와 점프 밟기는 MVP의 핵심 전투 수단이므로 판정이 명확해야 한다. 플레이어가 왜 피해를 주었는지 또는 받았는지 시각 효과와 사운드로 구분한다.

## 10. 몬스터 시스템

몬스터는 CSV 데이터로 관리한다. 기획자는 CSV 파일에서 체력, 피해량, 보상, 등장 구역을 수정할 수 있어야 한다.

몬스터 기본 속성:

- 몬스터 ID
- 이름
- 구역
- 최대 체력
- 접촉 피해
- 이동 속도
- 추격 범위
- 박치기 피해 배율
- 점프 밟기 피해 배율
- 경험치 보상
- 골드 보상
- 스폰 개수
- 보스 여부

몬스터 행동은 MVP에서 단순화한다.

- 기본 몬스터는 일정 범위 안에서 배회한다.
- 플레이어가 추격 범위 안에 들어오면 플레이어를 향해 이동한다.
- 보스는 넓은 추격 범위와 높은 체력을 가진다.

## 11. 보상 시스템

몬스터 처치 시 경험치와 골드를 즉시 지급한다. 보상량은 몬스터 CSV 데이터에 따른다. 같은 구역 안에서도 몬스터 타입별 보상 차이를 둘 수 있다.

## 12. 레벨 시스템

레벨은 1부터 10까지 제공한다.

| 레벨 | 필요 누적 경험치 | 비고 |
| --- | ---: | --- |
| 1 | 0 | 시작 레벨 |
| 2 | 50 | 쉬움 구역 반복 사냥 |
| 3 | 120 | 쉬움 구역 졸업 |
| 4 | 220 | 중간 구역 진입 |
| 5 | 360 | 중간 구역 안정화 |
| 6 | 540 | 어려움 구역 준비 |
| 7 | 760 | 어려움 구역 진입 |
| 8 | 1020 | 어려움 구역 안정화 |
| 9 | 1320 | 보스 구역 준비 |
| 10 | 1660 | MVP 최대 레벨 |

레벨업 효과:

- 최대 체력 증가
- 박치기 기본 피해 증가
- 점프 밟기 기본 피해 증가

MVP에서는 능력치 선택이나 스킬 해금 없이 자동 성장만 적용한다.

## 20. 테스트 항목

- 플레이어가 모든 방향으로 정상 이동하는가
- 점프가 지면에서만 실행되는가
- 박치기 중 몬스터에게 피해가 들어가는가
- 점프 하강 중 몬스터 상단을 밟으면 피해가 들어가는가
- 공격 상태가 아닐 때 몬스터 접촉 시 플레이어가 피해를 받는가
- 몬스터 체력이 0 이하가 되면 보상이 지급되는가
- 경험치가 누적되고 레벨 10까지 상승하는가
- 레벨 10 이후 경험치 표시가 깨지지 않는가
- 구역별 몬스터가 CSV 데이터에 맞게 스폰되는가
- 보스 처치 시 결과 화면이 정상 표시되는가
